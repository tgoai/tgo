TGO Plugin System allows developers to extend the functionality of the TGO customer service system through standardized interfaces. Plugins can add custom information display to the visitor panel, extend the chat toolbar, and integrate with third-party applications.

## What is a TGO Plugin

A TGO Plugin is a separate process that communicates with the TGO host program via **Unix Socket**.

### Core Advantages
- **Language Agnostic**：Supports any language that can handle Socket communication.
- **Isolation**：Plugin processes run independently, not affecting the stability of the main system.
- **Real-time Bidirectional Communication**：Supports the host pushing requests to plugins and plugins returning UI or instructions in real-time.

## Recommended Development Approaches

We highly recommend using the official **SDK** for development. The SDK encapsulates the underlying communication protocol, reconnection mechanism, and UI building logic, allowing you to focus on business logic.

## Core Concepts

The TGO Plugin System's interaction is based on the following core concepts:

- **Extension Point**：Plugins mount points, such as `visitor_panel` (Visitor Panel) or `chat_toolbar` (Chat Toolbar).
- **Template**：JSON data describing the UI structure. The SDK provides rich Builders (e.g., `Table`, `Form`, `Group`) to generate these templates.
- **Action**：Indicates the specific operation the host should perform, such as `OpenURL` (open a web page) or `ShowToast` (display a toast message).
- **Capability**：Plugins declare which extension points they support and their configuration details when registering.
